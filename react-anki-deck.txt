In `export default App`, what is `default`?	A keyword marking App as the module's default export. It is NOT a type. A file can have at most one default export but many named exports.
How do you import a default export?	Without braces: `import App from './App'`. You can rename it to anything on the import side.
How do you import a named export?	With braces: `import { paths } from './paths'`. The name must match the exported name (use `as` to rename).
Can you combine a default import and named imports?	Yes: `import React, { useState } from 'react'`. `React` is the default, `useState` is named.
What is a React component under the hood?	A plain function. Modern React has no classes, no instances, no mutable objects. A component takes props and returns a value (JSX).
What does JSX compile to?	Calls to `React.createElement(type, props, ...children)`, producing plain objects like `{ type, props, children }`. JSX is data, not an instance.
Who actually calls your component functions?	React's runtime. You never call `Component()` directly. Writing `<Component />` is just data telling React to call it later.
What happens when a component's props or state change?	React re-calls the component function. Each call returns a fresh JSX tree; React diffs it against the previous result and updates only the DOM that changed.
What is the `children` prop?	When you write `<Parent><Child/></Parent>`, JSX passes `<Child/>` as a special prop called `children`. The parent decides where (or whether) to render it.
How does component nesting work in React?	Function composition over a tree. Parents receive children as a value and render them somewhere inside their own returned JSX.
What is `createBrowserRouter(routes)`?	A factory function that takes a routes array and returns a router object bundling the routing table, URL matcher, and browser History API subscription.
Does RouterProvider intercept HTTP requests?	No. It's client-side. It listens to browser URL changes via the History API (pushState / popstate) and renders the matching route's element. No network involved.
How many router objects exist in a typical SPA?	One. Created once with `createBrowserRouter` and passed once to `<RouterProvider router={router} />`. Descendants access it via hooks like `useNavigate`, `useParams`, `useLocation`.
What is a routes array like `RouteObject[]`?	A plain array of `{ path, element }` objects — a routing table. `RouteObject[]` is just the TypeScript type annotation. Not a class.
What does `useTheme()` do?	Reads the theme object that `ThemeProvider` published to context. It does NOT create or copy anything — it returns a reference to the single theme created upstream.
What problem does React Context solve?	Prop drilling — passing a value through many intermediate components just so a deep child can use it.
What is a React Context under the hood?	A plain JavaScript object with special fields (a `$$typeof` symbol, `_currentValue`, a `.Provider`, a `.Consumer`). React uses the object's identity as a unique key.
What does `React.createContext(defaultValue)` return?	An object containing `.Provider` and `.Consumer` components plus an internal current value. The object itself acts as the key matching Providers to `useContext` calls.
What does `<SomeContext.Provider value={x}>` do internally?	React pushes `x` onto an internal stack keyed by the context object while rendering the children, then pops it after.
What does `useContext(SomeContext)` do?	Walks up the current component's fiber tree, finds the nearest Provider for that context object, and returns its `value`.
What is a fiber in React?	An internal node in React's component tree (one fiber per rendered component). Each fiber knows its parent, enabling upward walks for context lookup.
How does React match a `useContext` call to a Provider?	By object identity (`===`). The same context object must be passed both to create the Provider and to `useContext`.
How is React Context similar to Elixir's `Process.put/get`?	It's a form of dynamic scoping — but scoped to a render subtree rather than a process. Providers set values for descendants; useContext reads them without explicit prop passing.
What is prop drilling?	Passing a prop through many layers of components that don't use it themselves, just to get it to a deep child. Context avoids this.
Why is `<ThemeProvider>` placed as the outermost element?	So every descendant falls inside its context subtree. Only components rendered inside a Provider can read that Provider's value via the matching hook.
What happens when a Context Provider's `value` changes?	Every descendant component that reads that context re-renders with the new value.
What's the mental model for writing React code?	Write pure functions from props → JSX. React calls them at the right times and syncs the result to the DOM.
How is `MuiThemeProvider` implemented in one line?	Roughly: `<ThemeContext.Provider value={theme}>{children}</ThemeContext.Provider>`. Most "Provider" components are thin wrappers around a Context Provider.
How is `useTheme()` implemented in one line?	Roughly: `useContext(ThemeContext)`. Most custom `useX` hooks are named wrappers around `useContext`.
React component = ? (Elixir analogue)	Pure function returning a data structure (like returning a tagged tuple / AST node).
JSX element = ? (Elixir analogue)	A tagged tuple / AST node describing what to render.
`children` prop = ? (Elixir analogue)	A value (subtree) passed as an argument to a function.
Context + `useTheme` = ? (Elixir analogue)	Dynamic scope / `Process.get` — but over a render subtree instead of a process.
`RouterProvider` = ? (Elixir analogue)	Pattern-matching on the current URL and dispatching to the corresponding element.
Are React components called once or many times?	Many times. Every time props or state change, React re-calls the function. `useState` / `useTheme` work because React keeps state on the side and hands it back on each call.
What is a closure?	A function bundled with the variables from the scope where it was defined. Those variables stay alive as long as the function does, even after the outer scope has returned.
Give a minimal closure example in JavaScript.	`function makeCounter() { let c = 0; return () => ++c; }` — each call to `makeCounter()` creates a fresh `c`; the returned function keeps it alive.
Why do two `makeCounter()` calls produce independent counters?	Each call creates a new scope with its own `c`. The returned function closes over that specific `c`. Closures capture a scope, not a shared global.
Closure in Elixir terms?	`fn -> some_var end` — an anonymous function captures variables from its lexical environment. JavaScript does the same, but captured bindings can be mutated.
What role do closures play in Zustand?	The store's `state` and `listeners` live in the closure created when `create` ran. `setState`, `getState`, and your actions close over those variables and mutate them.
What is a React hook?	A function whose name starts with `use`, callable only inside a rendering component or another hook. It lets you hook into React's per-component state, context, effects, and subscriptions.
How does React track which state belongs to which `useState` call?	By call order. Per component, React keeps an ordered slot list; each hook call reads/writes its slot by index. No names — position is the key.
Why can't you call hooks inside `if` or loops?	Because React identifies hooks by call order. Skipping a call or adding one shifts every subsequent slot, so hooks read the wrong state or crash.
What is a "custom hook"?	A plain function named `useX` that calls other hooks. It inherits the call-order rules via its inner hook calls. Used to package reusable stateful logic.
What does `useSyncExternalStore(subscribe, getSnapshot)` do?	Lets React subscribe a component to an external data source. On each render it re-reads the snapshot; when the source notifies, React re-renders if the snapshot changed by `===`.
Elixir analogue for "hooks only work inside components"?	Like `self()` only being meaningful inside a process. React sets up an ambient "currently-rendering component" before calling your function; hooks reach into that.
Name four common built-in React hooks and what each does.	`useState` — per-component state. `useContext` — read nearest Provider value. `useEffect` — run fn after render, re-run on dep change. `useSyncExternalStore` — subscribe to external data.
What problem does Zustand solve over React Context?	Context re-renders every consumer on any value change. Zustand uses selectors, so a component only re-renders when its picked slice changes.
Is a Zustand store React-specific?	No. The core is plain JavaScript — a closure with `getState`, `setState`, `subscribe`. Usable in Node/tests without React. The React hook is a thin `useSyncExternalStore` wrapper on top.
What three methods does a raw Zustand store expose?	`getState()` — read current state. `setState(partial)` — update (shallow merge by default). `subscribe(listener)` — register a change callback; returns an unsubscribe function.
What does `create(initializer)` return in Zustand?	A custom React hook (e.g. `useProductStore`). The hook is also an object with `.getState`, `.setState`, `.subscribe` attached — so it works both inside and outside components.
Give a minimal Zustand store.	`const useCart = create((set) => ({ items: [], add: (x) => set((s) => ({ items: [...s.items, x] })) }));`
When does Zustand's initializer run?	Exactly once, at module-load time, when `create(...)` is called. NOT on every hook call and NOT on every re-render.
When does the Zustand hook run and what does it do?	On every render of a subscribed component. Reads current state, runs the selector, subscribes the component, returns the selected slice.
When does `set` run and what does it do?	When an action is invoked. Computes the new state (shallow merge unless `true` is the second arg), replaces the store's state, and notifies every listener.
What does a Zustand selector do?	Picks a slice from state. Components re-render only when the selector's return value changes by `===`. Without a selector, the whole state is returned and any change re-renders.
Does Zustand need a Provider wrapping your tree?	No. A store is a module-level singleton created in a JS closure. Components import the hook directly — no wrapping component needed.
Zustand vs GenServer (Elixir)?	store = process; `getState` = `GenServer.call(:get)`; `setState` = `GenServer.cast(:set)`; actions = handle_call/cast clauses; subscribe = Phoenix.PubSub. Zustand runs on a single event loop.
Why are Zustand action functions stored inside state?	So components can pull them with selectors and only one hook is needed. Action functions are stable references (not replaced on set), so selectors returning them don't cause re-renders.
Are "modules" or "components" subscribed to a Zustand store?	Components (specifically fibers). Each render that calls the hook registers a subscription; React re-renders that component when its selected slice changes.
What does `set({ items: [] })` vs `set({ items: [] }, true)` do?	Default is shallow merge — other fields are preserved. Passing `true` replaces state entirely. `storeRegistry` uses `true` so reset fully restores initial state.
Where does React Context appear in Zustand?	Nowhere. Zustand bypasses Context entirely; subscriptions go through `useSyncExternalStore` to the external JS store. That's why Context's re-render-all behavior doesn't apply.
What is the purpose of `storeRegistry.ts`?	Wraps zustand's `create` so every store is tracked in a module-level `Set`. `resetAllStores()` can then restore all of them to their initial state in one call.
What does `registerStore` add beyond plain `create`?	After creating the store it captures the initial state and adds a closure `() => store.setState(initialState, true)` to `storeResetFns`, making the store resettable.
Why `as typeof actualCreate` in storeRegistry?	TypeScript cast — tells the compiler our wrapper's type signature matches zustand's `create` exactly, so consumers can use it as a drop-in replacement.
Why the double call `actualCreate<T>()(initializer)` in zustand v5?	Curried form for TypeScript inference. `actualCreate<T>()` returns a function specialized to `T`; calling it with the initializer gives the store. A TS ergonomic trick.
Common uses of `resetAllStores()`?	Resetting state between tests, wiping all stores on user logout, a single "nuke" action in development.
Closure vs class for encapsulating state?	Both hide data behind an interface. Classes put state on `this`; closures put it in a captured scope. Zustand uses closures: no `new`, no `this`, just a function with private captured variables.
Timeline: what runs when with `useProductStore`?	Module load: `create(...)` runs once, initializer runs, state+actions stored. Render: hook reads state, subscribes. Action call: `set` updates state and notifies listeners → React re-renders subscribed components.
Closure example mirroring Zustand's internals.	`function makeStore(init) { let s = init; const ls = new Set(); return { get: () => s, set: (x) => { s = {...s, ...x}; ls.forEach(f => f()); }, sub: (f) => { ls.add(f); return () => ls.delete(f); } }; }`
What two things does MUI's DataGrid need?	`rows` (array of data objects) and `columns` (array of column descriptors). Everything else — sorting, pagination, filtering, virtualization — is handled internally.
What does DataGrid use to link a row object to a column cell?	The column's `field` value. `field: 'name'` means "for this column, read `row.name`". DataGrid uses `field` for display, sorting, and filtering.
Why does `GridColDef<Product>[]` give you compile-time checking?	The generic parameter tells TS that each `field` must be a key of `Product`. Typos like `field: 'naem'` become compile errors.
Difference between `width` and `flex` on a column?	`width: 80` is fixed pixels. `flex: 1, minWidth: 150` grows to fill remaining space (like CSS flexbox) but never smaller than `minWidth`. Use `flex` on the main column, `width` on the rest.
What does a column's `type` field control?	Sort behavior (numeric vs string), filter UI (number input / date picker / select), and default alignment. Values: `'string' | 'number' | 'date' | 'dateTime' | 'boolean' | 'singleSelect' | 'actions' | 'custom'`.
`valueFormatter` vs `renderCell` on a column?	`valueFormatter` returns a string for display — raw value stays unchanged (sorting still works numerically). `renderCell` returns JSX for custom components (chips, icons, buttons).
Why does DataGridPro need an explicit parent height?	Without `height` on a parent (or the `autoHeight` prop), the grid collapses to 0px and nothing is visible. Classic gotcha when a DataGrid appears blank.
In the wrapper `<DataGridPro defaults {...props} />`, why does spread come last?	Props from the spread override earlier defaults. Caller can pass `pagination={false}` and it wins. If spread came first, defaults would overwrite caller props — usually the wrong order.
What does `<DataGrid>` as a wrapper component achieve?	Bakes in house defaults (pagination, selection behavior, page sizes) so every table in the app looks/behaves the same. Same pattern as the `ThemeProvider` wrapper.
What does DataGrid use as a row's unique key by default?	A field literally named `id` on each row. If your rows use a different key, pass `getRowId={(row) => row.someKey}`.
What does `initialState` mean on DataGrid (vs `state`)?	`initialState` is uncontrolled — DataGrid manages it internally after first render. `state` is controlled — you'd track and sync it in React state yourself.
What is `useEffect(fn, deps)`?	A React hook that runs `fn` AFTER the component renders, and re-runs it whenever any value in `deps` changes. The escape hatch for side effects.
What does `useEffect(() => getProducts(), [getProducts])` do?	Runs `getProducts()` once on mount. Because `getProducts` is a stable reference from Zustand, the deps never change, so it doesn't re-run.
What happens if you omit the dependency array in useEffect?	`useEffect(fn)` without `deps` runs after EVERY render. If `fn` updates state, you get an infinite render loop.
Why call async loaders from `useEffect`, not directly in the component body?	The body runs during render. Updating state during render triggers another render → infinite loop. Effects run after the render commits, so updates are safe.
Elixir analogy for `useEffect`?	Like `GenServer.init/1` returning `{:ok, state, {:continue, :load}}` and doing the real work in `handle_continue`. Render = init; effect = work scheduled after the UI is committed.
How does the loading spinner in DataGrid get toggled?	Pass a `loading` boolean prop. The store flips `loading` to `true` before `await`, and back to `false` after the data is set. DataGrid renders an overlay while `loading` is true.
Data flow when the Product page mounts?	Mount → useEffect runs → store sets loading=true → awaits productsApi.getProducts() → sets products+loading=false → Zustand notifies → component re-renders → DataGrid paints rows.
Three-layer split for a DataGrid-based page?	columns.tsx = shape of the table (pure data); common/DataGrid.tsx = house defaults; ModulePage.tsx = fetch from store and pass rows/columns/loading as props.
Key insight about using libraries in React?	Learning a React library is mostly learning which props it takes. The `<X prop={value}>` contract is identical whether X is a `<div>` or a full-featured table.
